# handoffs フォルダ確認 — チェックリスト

**正本:** パス規則 · 終了ゲート · アーカイブ → [routing.md](routing.md)  
**用途:** スキル **確認モード**のみ。ファイルの移動・削除は行わない。

## 確認の段階（Tier）

| Tier | いつ | Read の範囲 | チェック ID |
|------|------|-------------|-------------|
| **0（横断索引）** | 「handoffs 全体」· プロジェクト未指定 · 「一覧」「状態」 | `handoffs/README.md`（あれば）· Glob `handoffs/*/HANDOFF.md` · 各 HANDOFF **先頭表のみ**（`| **最新セッション** |` · `| **状態** |`）· **セッション MD 全文は読まない** | I0 · H1 · H2（リンク文字列のみ）· 各 R1 は Tier-1 |
| **1（プロジェクト詳細）** | slug 指定 · `@.../handoffs/{project}/` · 「詳しく」「整合」「不整合」 | 当該 `{project}/` の H1〜T2（下表）· 必要時のみ最新セッション MD（T1 は §4 中心でよい） | 下表すべて |

**Tier 0 でやらないこと:** 全プロジェクトのセッション MD 全文 Read · T1/T2 · archive 内 MD の本文 Read。

**曖昧時（親スキル）:** 「全体の索引だけ」か「特定プロジェクトの詳細監査」かを **1 問**。

## 対象の決め方

| ユーザー指定 | Tier | スキャン範囲 |
|--------------|------|--------------|
| なし · 「handoffs 全体」· 一覧 · 状態 | **0** | 上表 Tier 0 |
| プロジェクト名 · `@.../workspace-layout/` | **1** | `c:/yk-memo/handoffs/{project}/` のみ |
| 特定セッション `@` | **1** | その MD + 同フォルダ HANDOFF · README · 兄弟（Glob） |

## チェック項目

### Tier 0 のみ

| ID | 確認内容 | ⚠ の例 |
|----|----------|--------|
| I0 | Glob の `{slug}` と `handoffs/README.md` 表の slug が一致 | README に無い project · README にあるがフォルダ無し |
| H1 | 各 `{project}/HANDOFF.md` が存在 | 無い |
| H2 | HANDOFF 先頭表の「最新セッション」リンク文字列が実在ファイル名と一致（Glob でファイル名のみ確認可） | 死リンク |
| X0 | 先頭表の **状態** 1 行を索引表に転記 | — |

**R1 / S1 / T1 等は Tier 1**（プロジェクト指定時）。

### Tier 1（プロジェクト詳細）

| ID | 確認内容 | ⚠ の例 |
|----|----------|--------|
| H1 | `HANDOFF.md` が存在する | 無い |
| H2 | HANDOFF の「最新セッション」リンクが開ける | 死リンク |
| R1 | `{project}/README.md` の「最新」と HANDOFF が **同一ファイル** | `_3` vs `_7` の不一致 |
| S1 | ルート直下のセッション MD 一覧（`archive/` 除く） | 最新以外が残存（**整理→archive 先行**未実施 · 要整理） |
| S2 | **最新は 1 本**（HANDOFF が指すもの） | 複数が「最新」扱いの記述 |
| A1 | `archive/{YYYY}/` の件数 · 直近の移動先 | 空のままルートだけ増殖 |
| L1 | 移行期 `YK_*_SESSION*` · `*_NEXT_SESSION.md` がルートに残っていないか | 重複ストック |
| T1 | 最新セッション MD の **§4**（+ テンプレ準拠時 §2 Git 表 · 依頼文）。全文不要なら §4 のみ Read / Grep | 空欄 · `TBD` |
| T2 | 秘密らしき行（`api_key` · `password` · 長いトークン） | 検出時は BLOCKER として報告 |

パス規則の詳細は [routing.md](routing.md) を Read すること。本ファイルに再掲しない。

## 確認モードでやらないこと

- セッション MD の新規 Write（終了モード）
- §4 のタスク実行（再開モード）
- archive 移動 · 削除（整理モード · ユーザー明示後）
- 「GO」と言って作業に入る（提案は箇条書きのみ）
- **Shell ツール** — Cursor の RUN 承認は主に Shell で発生する。確認は **Glob + Read**（+ Grep）で足りる
