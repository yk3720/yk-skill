# Personal Scheduler

ユーザーの生活リズムに合わせて、無理のないタイムスケジュールを提案し、外出先でもスマホで確認できるウェブページを生成・公開します。

## 依存ファイル

- `c:\yk-memo\03.schedule\` — スケジュール管理フォルダ（このフォルダ内の全 `.md` ファイルを読み込む）
- `references/template.html` — ウェブ表示用テンプレート（手帳・ノート風デザイン）

## ワークフロー

### Step 1: 管理フォルダの読み込み

`c:\yk-memo\03.schedule\` フォルダ内のすべての `.md` ファイルを読み込み、情報を統合する。

1. **メインスケジュール (`schedule.md`)**: 起床・就寝時間、固定の家事時間、各日の基本的なやりたいことリスト。
2. **個別スケジュール (例: `shipping.md`)**: 特定の業務（野菜出荷事務など）に特化した予定やタスク。
3. **優先順位の統合**: 全ファイルからタスクを抽出し、以下の優先順位に基づき1つのタイムスケジュールにまとめる。
   - **動的優先度**: タスク名の先頭に `[!!]` が付いているものは、最優先事項として固定時間の直後に配置する。
   - **基本順位**: 1. 野菜出荷, 2. 主婦業, 3. 勉強（農業・占星術）

### Step 2: スケジュール計算と最適化

現在の日付を中心に、計15日間を対象とするが、情報の粒度に強弱をつける。

1. **詳細スケジュール (今日〜3日後)**:
   - 30分刻みの厳密なプラン。
   - **バッファーの挿入**: 各タスクの間に15〜30分のバッファーを確保。
   - **移動の詳細化**: `locations.md` の難易度設定に基づき、乗り換え時間を動的に調整する。
2. **サマリスケジュール (4日後〜14日後)**:
   - 重要な予定と移動のみを反映した、1時間単位のざっくりとしたプラン。

- **時間超過の調整**: 優先順位（動的含む）に基づき、勉強時間を短縮するか翌日に回す。
- **日を跨ぐ調整**: 終わらなかったタスクが翌日の「やりたいことリスト」に未完了として残っている場合、翌日のスケジュールに組み込む。

### Step 3: 管理ファイルの更新

各日付のセクション（`## YYYY-MM-DD`）にタイムスケジュールを書き込む。

### Step 4: ウェブページの生成（24hバーチカル + モバイル最適化）

`references/template.html` を使い、以下のHTML群を `output/` 内に生成する。

1. **`index.html` (直近サマリー)**:
   - 直近3日間の主要な予定を表示。

2. **個別ページ (`YYYY-MM-DD.html`)**:
   - **24時間バーチカル表示**: 5:00から翌5:00まで。
   - **現在時刻ジャンプ機能**: ページを開いた際、またはボタン押下で現在の時刻の行にスクロールする。
   - **タスク完了連携**: 各タスクに「完了用MDをコピー」ボタンを配置。タップすると `[x] タスク名` がコピーされ、MDへのフィードバックを容易にする。
   - **公式時刻表の反映**: 移動がある場合、必ず `WebSearch` で最新の時刻表を確認。

3. **`archive.html` (月間アーカイブ)**:
   - 月間カレンダー形式で、生成済みのすべての日付へのリンクを表示する。
   - これにより、過去のスケジュールをどこまでも遡れるようにする。

### Step 5: 公開と履歴の維持

`output/` 内の既存のHTMLファイルを削除せず、新しいファイルを順次追加して `surge.sh` に公開する。
これにより、URLをブックマークしておけば、いつでも過去から未来までの全履歴を閲覧できる。

```powershell
npx --yes surge output --domain personal-schedule-yk.surge.sh
```

### Step 6: 完了報告

公開されたURLをユーザーに伝え、スケジュールのポイント（どこにバッファーを入れたか、何を調整したか）を説明する。

## 禁止事項

- ユーザーの就寝時間を削ってスケジュールを組まない。
- 3時間以上の連続作業を入れない（必ず休憩を挟む）。
- バッファーをゼロにしない。
