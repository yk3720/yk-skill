# Personal Scheduler

ユーザーの生活リズムに合わせて、無理のないタイムスケジュールを提案し、外出先でもスマホで確認できるウェブページを生成・公開します。

## 依存ファイル

- `c:\yk-memo\00.ai-driven-school\schedule.md` — スケジュール管理のメインファイル（SSoT）
- `references/template.html` — ウェブ表示用テンプレート（手帳・ノート風デザイン）

## ワークフロー

### Step 1: 管理ファイルの読み込み

`c:\yk-memo\00.ai-driven-school\schedule.md` を読み込み、以下の情報を把握する。

1. **基本稼働時間**: 起床時間、就寝時間、固定の家事時間など
2. **やりたいことリスト**: ユーザーが追記した、特定の日のタスク（`## YYYY-MM-DD` セクション内）
3. **優先順位**: 1. 野菜出荷, 2. 主婦業, 3. 勉強（農業・占星術）

### Step 2: 前後1週間のスケジュール計算

現在の日付を中心に、前7日間、今日、後7日間の計15日間を対象とする。

- **バッファーの挿入**: 各タスクの間に15〜30分のバッファーを確保。
- **時間超過の調整**: 優先順位に基づき、勉強時間を短縮するか翌日に回す。
- **日を跨ぐ調整**: 終わらなかったタスクが翌日の「やりたいことリスト」に未完了として残っている場合、翌日のスケジュールに組み込む。

### Step 3: 管理ファイルの更新

各日付のセクション（`## YYYY-MM-DD`）にタイムスケジュールを書き込む。

### Step 4: ウェブページの生成（5日分サマリー + 各日詳細）

`references/template.html` を使い、`index.html` および各日付のHTMLを `output/` 内に生成する。

1. **`index.html` (5日分表示)**:
   - **対象範囲**: 現在の日付の1日前から3日後までの計5日間。
   - **レイアウト**: 
     - PC: 5つのカラム（またはカード）を横並びで表示。
     - スマホ: 縦にスタックして表示。
   - **内容**: 各日の「やりたいことリスト」と「主要なタイムスケジュール」のダイジェストを表示する。

2. **個別ページ (`YYYY-MM-DD.html`)**:
   - 対象の15日間それぞれの詳細HTMLを生成する。
   - 1日分の全スケジュールとメモを表示する。
   - **ナビゲーション**: ページ内に「前日」「翌日」および「5日分一覧（index.html）」へのリンクを生成する。

### Step 5: 公開（デプロイ）

`output/` ディレクトリ全体を `surge.sh` に公開する。

```powershell
npx --yes surge output --domain personal-schedule-yk.surge.sh
```

### Step 6: 完了報告

公開されたURLをユーザーに伝え、スケジュールのポイント（どこにバッファーを入れたか、何を調整したか）を説明する。

## 禁止事項

- ユーザーの就寝時間を削ってスケジュールを組まない。
- 3時間以上の連続作業を入れない（必ず休憩を挟む）。
- バッファーをゼロにしない。
