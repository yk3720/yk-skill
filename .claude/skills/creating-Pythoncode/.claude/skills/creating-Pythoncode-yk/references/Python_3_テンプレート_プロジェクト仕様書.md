# プロジェクト仕様書：[アプリ名]

## 1. 概要・目的
- **役割**: 本アプリが解決する課題と、提供する核心的価値を記述せよ。
- **ターゲット環境**: OS（Windows 10/11）、Python バージョン（.python-version）、配布形式（exe / uv 仮想環境）。

## 2. UI・操作仕様（物理的確定性）
- **レイアウト**: 配置、サイズ、配色（黄色 `#FFD700` 等）を解釈の余地なく確定させよ。
- **操作フロー**: ユーザーの入力から出力に至る導線を「直感的」に定義せよ。

## 3. コアロジック・計算仕様
- **処理アルゴリズム**: 条件分岐、データ変換、外部連携の論理を数学的・論理的に記述せよ。
- **内部資産**: ファイル名体系、ID体系、定数配列の整合性を確定させよ。

## 4. 極限境界値への対処 (Edge Cases)
- **物理적遮断・入力の暴力**: 通信切断や異常なデータ入力（巨大、空、不正型）への対処方針を記述せよ。
- **時間の歪み・権限の壁**: タイムアウトやアクセス権限不足時の挙動を定義せよ。

## 5. ユーザーガイド
- **基本的な使い方**:
  1. 起動（exe実行、または `uv run main.py`）。
  2. [主要操作1] の実施。
  3. 実行結果の確認（`output/` 等）。
- **困ったときは**: 代表的なエラー（権限不足、入力不備）への「ガイダンス型」対処法。

## 6. 開発スタック・環境構成
- **管理ツール**: `uv` (pyproject.toml / uv.lock)
- **品質維持**: `Ruff` (Linter/Formatter), `mypy` (型検査), `pytest` (自動テスト)
- **主要ライブラリ**: `CustomTkinter` (GUI), `loguru` (Logging), `Pydantic` (Schema)

## 7. 物理的ディレクトリ構成
```text
project_root/
  ├── app/                # ソースコード本体
  │   ├── core/           # 業務ロジック（UIを知らない、Schemasを参照）
  │   ├── ui/             # GUI定義（Core/Schemasを参照）
  │   ├── schemas/        # データ型定義（Pydantic等。中立地帯）
  │   └── utils/          # 共通ユーティリティ
  ├── tests/              # 自動テスト (pytest)
  ├── 仕様・管理/         # 必須3文書 (Spec, ADR, Test)
  ├── output/             # 生成物出力
  ├── temp/               # 一時ファイル（clean.py の対象）
  ├── main.py             # エントリポイント
  ├── pyproject.toml      # 依存関係・ツール設定
  └── clean.py            # 環境浄化スクリプト
```
