## 参照ルール（重要）

### 参照を追加してよい条件（「未参照ファイル」を増やしすぎない）

次のいずれかに当てはまるときだけ、`SKILL.md` からリンク参照を追加します。

- **手順のゲート**（監査/品質/締切など）に直結する
- **頻出する判断基準**（毎回の作業で迷いが出る）を固定したい
- **テンプレートの利用**が作業の標準手順に含まれる

逆に、以下は原則として `SKILL.md` から参照しません（この `references/` に置くだけ）。

- 一部の案件でしか使わない詳細ノウハウ
- まだ確定していない検討メモ
- 参照頻度が低い補助資料

### SSoT（矛盾防止）

- **詳細は `references/` に1箇所だけ**置く（重複記述でズレを作らない）
- `SKILL.md` は「短い指示＋参照リンク」で構成する
- 参照先を更新したら、`SKILL.md` 側で **リンク切れ/手順矛盾** が起きていないか確認する

### リンク整合のチェック（運用）

少なくとも次を満たすことを確認します。

- `SKILL.md` から参照されているファイルが **実在**する
- `references/` に置いたファイルのうち、**必読/準必読/補助** の位置づけがこのREADMEで説明できる

## 参照不要（保管物）

この `references/` には、`SKILL.md` から直接リンク参照しないファイルが含まれることがあります。これらは「未参照＝不備」ではなく、運用上の保管物です。

- `.gitignore`：このフォルダ配下の除外設定（参照して読む必要は基本なし）

このREADMEは「この `references/` が何のためにあるか」を説明するSSoTです。別プロジェクトの説明を流用しないでください。
