# Git 履歴保護規律 (Git History Protection Rule)

## ■ 背景と目的
本リポジトリでは、ナレッジ/規律の変更履歴は **Git を唯一のSSoT** として管理する。
従来の「rev番号ファイルを積層する運用（二世代維持ルール）」は採用しない。

## ■ 厳禁事項（Git前提）
1. **履歴の破壊**:
   - `git push --force` 等による履歴改ざんを行わない。
   - 既存コミットの改変（rebaseの乱用等）で監査可能性を落とさない。
2. **無断の破壊的操作**:
   - AIが独断で `reset --hard` や大規模削除を実行しない（必要なら差分提示→ユーザー判断）。

## ■ 実行プロトコル（更新方法）
1. **通常更新**:
   - ナレッジ/規律は同一ファイルを更新し、Git差分で追跡する（rev番号の新規ファイル作成で管理しない）。
2. **大きな方針転換**:
   - 変更理由・影響をADR等に記録し、履歴で追跡できるようにする。

## ■ 自己監査要件
- **検挙（実装前）**: 破壊的コマンド（force push / hard reset）を要求・実行していないか。
- **検挙（報告前）**: 変更理由と影響が差分から追える状態か。
