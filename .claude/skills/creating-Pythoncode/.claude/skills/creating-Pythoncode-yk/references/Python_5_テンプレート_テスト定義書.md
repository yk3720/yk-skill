# テスト定義書：[アプリ名]

## 1. 正常系テストケース
| ケースID | 検証項目 | 入力値・操作 | 期待される結果 (Expected) |
| :--- | :--- | :--- | :--- |
| TC-001 | 起動確認 | exeをダブルクリック | GUIが正常起動し、ステータスバーに「準備完了」と表示 |
| TC-002 | データ処理 | 有効な入力データを読み込み | 出力フォルダに[ファイル名]が生成されること |

## 2. 異常系・境界値テストケース
| ケースID | 検証項目 | 入力値・操作 | 期待される結果 (Expected) |
| :--- | :--- | :--- | :--- |
| TC-E01 | 入力の暴力 | 空のファイルを選択 | ガイダンス型通知にて「ファイルが空です」と警告 |
| TC-E02 | 権限の壁 | 書き込み禁止フォルダ指定 | 【状況】＋【原因】＋【具体的アクション】が表示されること |

## 3. 静的データ整合性監査
- **ID連番性**: 全ての雛形データIDが重複なく連続していることを監査せよ。
- **参照整合性**: 定数配列のインデックスが論理的に正しいか監査せよ。

## 4. 自動監査・品質儀式 (Standard Stack)
- **Ruff (Linter/Formatter)**: `uv run ruff check` および `format` で警告ゼロを確認。
- **mypy (Static Type Check)**: `uv run mypy app/` でエラーゼロを確認。
- **pytest (Unit Test)**: `uv run pytest` ですべてのテストが PASS すること。

## 5. 環境浄化監査
- **clean.py 実行**: 一時ファイル（`temp/`, `__pycache__` 等）が物理的に抹消されているか。
- **TODO スキャン**: コード内に `TODO:` や `FIXME:` が残存していないか。
