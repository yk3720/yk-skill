今日できたこと

「まず動かすこと」を優先し、LINEへの自動送信を最初のゴールに設定。6 つのステップを今日 1 日で完走しました。

1
LINE公式アカウント作成

「業務報告」アカウントを新規作成。GASから操作するための入り口を用意した。

2
Messaging API 設定

LINE Developers Console で Messaging API を有効化。チャネルアクセストークンを取得し、GASから LINE を操作できる状態にした。

3
Google スプレッドシート作成

帰宅中にスマホで入力できる進捗入力フォームをスプレッドシートで作成。

4
GAS スクリプト作成

データ読み込み → 進捗サマリー生成 → LINE 送信 の一連の自動処理を実装した。

5
動作確認

テスト送信を実行 → LINE にレポートが届くことを確認。ゴール達成。

6
自動トリガー設定

毎日指定時刻に自動でレポートが届く設定を完了。これで翌日から手作業ゼロに。

アカウント・ID の管理が思った以上に大変

今回だけでも一気に増えました。自動化を進めるほどアカウントや API キーが増えていくため、パスワードマネージャーなどで一元管理する仕組みを早めに作ることが重要だと感じました。

GAS の時間トリガーは「分」まで指定できない

GAS（Google Apps Script）の時間トリガーは、1 時間単位での設定になっています。「19:30 に実行」という指定はできず、「19 時〜20 時の間でランダムに実行」という形式です。
