# 技術用語 → やさしい言葉 変換辞書

## 変換の原則

1. **機能で命名する**: 「何をするものか」で名前をつける
2. **制御・FA経験に紐づくたとえを優先**: PLC・ラダー・センサー・シリンダ等で連想できるものを使う
3. **初出時に解説ボックス**: 最初に出てきたとき必ず説明する

---

## 制御・FA用語とのたとえ対応表（汎用）

制御設計の経験をベースに、他分野の技術を理解するときの橋渡しに使う。

| 他分野の概念 | 制御・FAでたとえると |
|------------|------------------|
| イベント駆動 | センサーONをトリガーにシリンダが動く（割り込み処理と同じ） |
| 非同期処理 | 複数軸を同時に動かしながら完了を待つ（マルチタスクPLC） |
| キュー（待ち行列） | 工程バッファ（前工程の完了を待つ仕掛り品の列） |
| 状態機械（ステートマシン） | ラダーのステップシーケンス（STEP管理） |
| ポーリング | PLCがセンサー状態を毎スキャン確認すること |
| インターフェース | 入出力の仕様書（何を入れると何が返るかを決めた約束） |
| API | 設備メーカーの通信コマンド表（決まった書式で命令する） |
| バッファ | 受信バッファと同じ（データをためておく一時領域） |
| スレッド | 複数タスクの並列実行（PLCのマルチタスク機能） |
| スタック | PLCのネスト呼び出し（関数の呼び出し履歴） |
| コールバック | 完了割り込み（処理が終わったら呼んでもらう仕組み） |

## よく使う変換表（分野別）

### プログラミング基礎

| 技術用語 | やさしい言葉 | たとえ（制御・FA寄り） |
|---------|------------|-------------------|
| 変数 | 名前付きの箱 | PLCのデータレジスタ（D100など）と同じ |
| 関数 | 処理のまとまり | PLCのファンクションブロック（FB）と同じ |
| 引数 | 渡す情報 | FBに与えるIN変数 |
| 戻り値 | 返ってくる結果 | FBから得るOUT変数 |
| ループ | 繰り返し処理 | PLCのFOR命令と同じ |
| 条件分岐 | 条件によって処理を変える | ラダーのIF文・CASE文 |
| クラス | 設計図 | FBの型定義（インスタンスを複数作れる） |
| インスタンス | 設計図から作った実物 | FBを複数コピーして使うのと同じ |
| ライブラリ | 再利用できる部品集 | PLCのライブラリFB（メーカー提供の標準FB） |
| デバッグ | バグを見つけて直す | PLCのモニタリング・ブレークポイント |

### ネットワーク・通信

| 技術用語 | やさしい言葉 | たとえ（制御・FA寄り） |
|---------|------------|-------------------|
| プロトコル | 通信の約束事 | Modbus・PROFIBUS等の通信規格と同じ |
| ポート | 窓口番号 | PLCのチャンネル番号 |
| パケット | データの小包 | 通信フレーム（ヘッダ＋データ＋チェックサム） |
| ソケット | 通信の接続口 | シリアル通信ポートの接続と同じ |
| クライアント | お願いする側 | マスター局（Modbus Master等） |
| サーバー | お願いされる側 | スレーブ局（Modbus Slave等） |
| レイテンシ | 応答の遅れ | 通信サイクルタイムと同じ概念 |
| スループット | 単位時間あたりの処理量 | 設備のタクトタイムに相当 |

### データ・構造

| 技術用語 | やさしい言葉 | たとえ（制御・FA寄り） |
|---------|------------|-------------------|
| 配列 | 順番付きリスト | PLCのデータブロック（連続アドレス） |
| 構造体 | 関連データのまとまり | PLCの構造体型（STRUCT）と同じ |
| JSON | データの書き方の一つ | 設備仕様書のフォーマットと同じ（ルールに従って書く） |
| データベース | 大量データを整理して保存する仕組み | トレーサビリティシステムの生産実績DB |
| ログ | 動作記録 | アラーム履歴・オペレーションログと同じ |

### セキュリティ・権限

| 技術用語 | やさしい言葉 | たとえ（制御・FA寄り） |
|---------|------------|-------------------|
| 認証 | 本人確認 | PLCのパスワードロック |
| 認可 | 権限の付与 | オペレータ権限・管理者権限の区別 |
| 暗号化 | 読めない形に変換 | 通信の暗号化（SSLと同じ概念） |

### AI・機械学習

| 技術用語 | やさしい言葉 | たとえ（制御・FA寄り） |
|---------|------------|-------------------|
| モデル | 判断の仕組み | 品質検査機の判定ロジック |
| 学習データ | お手本のサンプル | ティーチングデータ（良品・不良品サンプル） |
| 推論 | 学習結果を使って判断する | 検査機が画像を見て良否判定するのと同じ |
| 過学習 | お手本に合わせすぎて応用が利かない状態 | 特定ワークにしか対応できない専用治具 |
| パラメータ | 動作を決める数値 | サーボアンプのゲイン・フィルタ設定値 |
