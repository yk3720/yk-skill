デザインの4大原則

接近（Proximity）・整列（Alignment）・反復（Repetition）・対比（Contrast）

1

ひとことで言えば

デザインの4大原則（PARC / CRAP）とは、
「なぜこの画面は読みにくいのか」を4つの軸で診断し、
修正する判断フレームワークである。

対象読者

UIを実装するエンジニア。デザイナーとの協業経験があるが、デザイン原則を体系的に学んだことはないケース。

カバー範囲

レイアウト・タイポグラフィ・余白設計の基礎。UXリサーチやユーザビリティテストは範囲外。

2

なぜ生まれたのか

背景：DTPの民主化（1990年代前半）

Macintoshの普及とDTP（Desktop Publishing：デスクトップ パブリッシング）ツールの登場により、「デザイン専門教育を受けていない人」が チラシ・スライド・Webページを自分で作る局面が急増した。 ツールは使えるようになったが、「どう配置すれば読みやすいか」の判断基準を持つ人は少なかった。

著者のロビン・ウィリアムズ（Robin Williams）は、この状況に対し「ビジュアル素人が今日から使える、少数の反復可能なルール」を 体系化した。それが1994年初版の『The Non-Designer's Design Book（ノンデザイナーズ・デザインブック）』で提示した4原則である。

なぜ人間に効くのか — 知覚的根拠（ゲシュタルト心理学）

4原則が「なんとなく正しい」のではなく、実際に機能するのは人間の知覚が構造を自動補完するという認知特性に根ざしているからだ。 1910〜1930年代に発展したゲシュタルト心理学（Gestalt Psychologie）は、 「人間は個々の要素の集合ではなく、まとまり（全体）として視覚情報を認識する」という原則を体系化した。

接近 ← 近接の法則（Law of Proximity）

近いものは「同じグループ」として知覚される

反復 ← 類似の法則（Law of Similarity）

見た目が似たものは「同じ種類」として知覚される

整列 ← 連続の法則（Law of Continuity）

連続した線・方向に沿うものは「一続き」として知覚される

対比 ← 図地の法則（Figure-Ground）

差が大きいほど「前景（主役）」が際立つ

4大原則はゲシュタルト知覚の事実を「レイアウト操作の手順」に翻訳したものであり、原則を守ると「なぜか読みやすい」ではなく「知覚特性に乗っているから速く理解できる」という因果がある。

解きたかった「あるある」課題

全要素を等間隔に並べれば整うという思い込み
何がひとかたまりなのか見た目から判断できない
フォントの種類・罫線・装飾が多すぎてごちゃつく
同じ階層なのに見た目がバラバラで一貫性がない
3

まず理解する3つ

A

4原則は「視覚的な文法」である

英語の文法が「動詞の位置」で意味を決めるように、デザインは「要素の位置・差・一貫性」で情報構造を伝える。 コードを書く前に「この配置は何を読者に伝えるか」を問うのが4原則の使い方。

B

反復と対比はセットで機能する

反復（同じパターン）が土台にあって初めて、対比（差）が「意図的な強調」として読める。 反復なしの対比は「バラつき」にしか見えない。この2原則は常に一緒に設計する。

C

余白は「空き」ではなく「境界の情報」

接近と整列を正しく適用すると、余白が「ここで意味のグループが切れる」という論理的な境界を担う。 デフォルトの margin を全て8pxに統一するだけでは不十分で、 グループ内（密）・グループ間（疎）を意図的に設計する必要がある。

4

4原則の詳細

用語：PARC / CRAP

4原則の頭文字を並べた略称。PARC = Proximity, Alignment, Repetition, Contrast の順。 同じ4語を並べ替えた CRAP（Contrast, Repetition, Alignment, Proximity）も広く使われる ―ロビン・ウィリアムズ自身がユーモアとして紹介した覚え方。

4原則の作業順序（典型的なワークフロー）

① 接近

情報の塊を決める

② 整列

各塊に軸を通す

③ 反復

全体に合図を敷く

④ 対比

視線の入口を作る

接近（Proximity）

P of PARC

定義

関連する項目同士を近くに、関連が弱い（または無関係な）もの同士は遠ざけることで、 視覚的な「まとまり（グループ）」を作る原則。

効果

  • ・情報が「塊」として素早く理解される
  • ・余白が意味を帯びる（論理境界になる）
  • ・スキャン速度が上がる

よくある誤用

  • ・全要素を「同じ間隔」で並べる
  • ・詰めすぎで「混線」状態になる
  • ・罫線で無理に囲い、ノイズを増やす

たとえ話：仕分けセンターの棚

宅配便の仕分けセンターでは、同じ地域への荷物をひとつの棚にまとめる。 「A棚 = 東京23区」「B棚 = 大阪市内」のように物理的な近さで「仕分け」を表現する。 同じ考え方で、画面上のラベルと値を同じ「塊」として配置し、他の塊とは間隔を広げる。

実装例：フォームのグルーピング

Bad: 全て8px間隔

Good: グループ内密・グループ間疎

整列（Alignment）

A of PARC

定義

各要素の位置を、見えている・見えていないにかかわらず共通の軸（辺・ベースライン・グリッド線）に揃える原則。 「偶然そろった」ではなく、一貫した基準にそろえるという意図が重要。

効果

  • ・視覚的な「静かな秩序」が生まれる
  • ・視線の導線が予測可能になる
  • ・「どこに次の情報があるか」が分かる

よくある誤用

  • ・要素ごとに違う左マージンが累積する
  • ・画像と文字の開始軸がバラつく
  • ・グリッド未設定で数pxのズレが多発

たとえ話：方眼紙のマス目

数学の方眼紙では、グラフの点を「縦横のマス目という共通の軸」に乗せることで、 複数の点の位置関係が即座に読める。この「共通の軸」がなければ 点の間の関係は座標なしの平面に打った点と同じ―関係性が読めない。 デザインのグリッドも同様で、「見えない縦軸（垂直グリッド線）」に各要素の左端を揃えると、 読者は「次の要素がどこに来るか」を無意識に予測しながら視線を動かせる。

用語：ベースライン（Baseline） ― テキストの文字が載る仮想の水平線。Flexboxの align-items: baseline は複数要素のベースラインを揃えるプロパティ。

整列の失敗と修正

Bad: 軸がバラバラ

Good: 左軸で統一

反復（Repetition）

R of PARC

定義

色・形・太さ・角丸・行間・余白段階など、意図した視覚要素を全体に繰り返し使用し、 「同じ家系」の統一感を作る原則。コードで言えば、コンポーネントの単一化に相当する。

効果

  • ・学習コストが下がる（2ページ目から速い）
  • ・ブランド的な統一感が生まれる
  • ・対比（例外）が意図として読める土台になる

よくある誤用

  • ・似せたつもりが微妙に違う（色・余白・フォント）
  • ・コピペ由来の例外CSSが点在する
  • ・反復だけで単調になる（対比が必要）

たとえ話：工場の組み立てライン

自動車工場の組み立てラインでは、各工程で使うボルトの規格・トルク・工具が統一されている。 「工程Aだけ違う規格のボルト」があると、品質検査と修正コストが跳ね上がる。 UIも同じで、border-radius: 8px を「このカードだけ6px」にすると 保守コストと統一感の両方が壊れる。デザイントークンとして変数化するのが解決策。

用語：デザイントークン（Design Token） ― 色・スペーシング・フォントサイズ等の視覚的な値を変数として定義したもの（例：--color-primary: #3B82F6）。 反復を機械的に保証するための仕組み。Tailwindの theme 設定が代表例。

対比（Contrast）

C of PARC

定義

大きさ・明度・太さ・色面積・密度などで差を作り、重要度・種類・階層を一目で分ける原則。 差が曖昧なら対比ではない―はっきり違わせるか、全く同じにするかの二択。

効果

  • ・視線の入口（最初に見る場所）が確定する
  • ・スキャン（ざっと読み）が可能になる
  • ・情報の階層が一目で伝わる

よくある誤用

  • ・差が弱い（全部が「中の強さ」で平坦）
  • ・強調が多すぎて視線の入口が消える
  • ・コントラスト比不足（アクセシビリティ違反）

たとえ話：オーケストラの指揮

オーケストラで全楽器が常にフォルテ（最大音量）で演奏し続けたら、 ソロパートは埋もれてしまう。ピアニッシモ（弱音）があるからフォルテが際立つ。 UIでも「全要素を目立たせる」と結果的に「何も目立たない」画面になる。 主役の行動ボタン（CTA）を1つ決め、それ以外を意図的に「弱音」にすることが対比の本質。

用語：WCAG（Web Content Accessibility Guidelines：ウェブコンテンツ アクセシビリティ ガイドライン） ― W3Cが定めるWebアクセシビリティの国際標準。テキストと背景のコントラスト比を 4.5:1以上（標準テキスト）にすることを要求している。
計算式: 比 = (L1 + 0.05) / (L2 + 0.05)（L1 が明るい方、L2 が暗い方の相対輝度）。 相対輝度はsRGBの非線形補正値から求める知覚工学的な定義であり、「なんとなく」ではなく人間の輝度感度に基づいた数値。 WebAIM Contrast Checker で即座に確認可能。

原則同士の関係性

接近 ↔ 整列

接近でグループを作り、その内部を整列で揃える二段構成が基本。フォーム・カード群・ダッシュボードで頻出。

反復 ↔ 対比

反復が「基本パターン」を作り、対比が「例外・強調」を生む。反復なき対比はただのバラつきになる。

整列 ↔ 反復

同じグリッド段階・行長・基準幅を繰り返すと、整列の維持コストが大幅に下がる。

接近 ↔ 対比

グループ内の差（ラベルと値）に弱い対比、グループ間には強い距離（空間的な対比）を使う合わせ技が多い。

6

エンジニアがはまりやすい落とし穴

余白のマジックナンバー化（反復の崩壊）

margin: 4px、 5px、 7px が混在している状態。 デザイントークンを使わずにその場で調整した結果、視覚的な統一感が失われる。

対策: 4の倍数（4/8/12/16/24/32px）などの段階的なスペーシングスケールを変数化し、そこから外れない。Tailwindなら space-y-2（8px）/space-y-4（16px）の段階を守る。

「とりあえず中央揃え」でスキャン性が死ぬ（整列の誤用）

テキストを全て text-center にすると、 見出し・本文・ラベルの左端が揃わず視線の導線がなくなる。 ヒーロー見出しなら中央でもよいが、情報量が多いコンテンツには左揃えが原則。

対策: 本文・フォーム・リストは左揃えを基本とし、中央揃えはコンパクトな要素（ボタン・バッジ・ヒーローコピー）に限定する。

罫線・シャドウで「接近」の代替をしようとする

グルーピングが弱い箇所を border や box-shadow で囲って補おうとする癖。 結果として罫線が増え、視覚的ノイズが積み上がる。接近（余白の差）で解決できるなら、罫線は不要。

対策: まず余白の「密・疎」で解決を試みる。どうしても境界が必要な場合のみ、細い罫線か背景色の変化を使う。

「強調インフレ」―対比が多すぎて何も目立たない

太字・赤字・下線・アイコン・背景色をページ全体に使い散らかした状態。 全ての要素を強調すると、優先度の差が消えてしまう。

対策: 1画面に「最も重要な視線の入口」を1つだけ決め、それだけを強い対比にする。その他は反復でトーンを下げる。

低コントラスト比によるアクセシビリティ違反

グレーのプレースホルダーテキスト、薄いサブテキスト、ライトモードの薄いボーダーなど。 視覚的に「洗練されたデザイン」に見えても、WCAG 2.1 AA基準（4.5:1）を下回るケースが多い。

対策: Chrome DevToolsのコントラストチェッカー、または WebAIM Contrast Checker で実装時に必ず確認する。

7

専門用語集

PARC / CRAP デザイン4原則の略称

Proximity, Alignment, Repetition, Contrast の頭文字。CRAPは同じ4語を並べ替えたユーモアある別名。

ゲシュタルト原則（Gestalt Principles）

人間が視覚情報を「まとまり」として認識する知覚心理学の法則群。4大原則はこの知覚事実を実践的操作に落とし込んだもの。 → 関連: 接近・反復

デザイントークン（Design Token）

色・スペース・フォントサイズ等の視覚値を変数として一元管理する仕組み。反復原則をコードレベルで保証する手段。 → 関連: 反復

ベースライン（Baseline）

テキストの文字が乗る仮想の水平線。CSSの align-items: baseline はこの軸を揃えるプロパティ。 → 関連: 整列

WCAG（Web Content Accessibility Guidelines） ウェブコンテンツ アクセシビリティ ガイドライン

W3Cが定めるWebアクセシビリティ国際標準。AA基準でテキストコントラスト比4.5:1以上を要求。 → 関連: 対比

ネガティブスペース（Negative Space）

要素と要素の間、または要素の外側の「空き」領域。接近原則では余白が論理的な境界情報を担う。 → 関連: 接近

CTA（Call to Action：行動喚起）

ユーザーに特定の行動（クリック・購入・登録）を促すUI要素。対比原則で最も強調すべきターゲット。 → 関連: 対比

グリッドシステム（Grid System）

画面を等分した仮想の縦/横線網。整列原則の実装基盤。CSSの display: grid で実現。 → 関連: 整列・反復

フィッツの法則（Fitts' Law）

ターゲット（ボタン等）への到達時間は「サイズが大きく、近いほど短い」という到達容易性の法則。対比・整列と相性が良い。 → 関連: 対比・整列

DTP（Desktop Publishing） デスクトップ パブリッシング

PCを使って印刷物・出版物のレイアウト・デザインを行うこと。4大原則が生まれた1990年代の技術的背景。

ヒーローセクション（Hero Section）

Webページの最上部に配置される大型の視覚的ブロック。キャッチコピー・画像・CTAボタンが置かれることが多い。落とし穴セクション内「ヒーロー見出し」「ヒーローコピー」はここに置かれるテキスト要素を指す。 → 関連: 対比

行長（Measure）

1行に収まるテキストの横幅・文字数。CSSでは max-width で制御する。長すぎると視線の折り返しコストが増え、短すぎると読みにくい。一般に 45〜75文字が読みやすい行長とされる。 → 関連: 整列・反復

タイプスケール（Type Scale）

見出しH1・H2・本文・キャプションなど、テキスト階層ごとのフォントサイズを比率で定義したもの（例：1.25倍または1.5倍の等比数列）。反復・対比の両方に関わる中核的な設計パラメータ。 → 関連: 反復・対比

Atomic Design（アトミックデザイン）

Brad Frost が提唱したUI設計方法論。Atom（原子：ボタン・入力欄）→ Molecule（分子：フォームグループ）→ Organism（有機体：ヘッダー）→ Template → Page という5階層でコンポーネントを体系化する。反復原則をコードアーキテクチャレベルで実現する手法。 → 関連: 反復
