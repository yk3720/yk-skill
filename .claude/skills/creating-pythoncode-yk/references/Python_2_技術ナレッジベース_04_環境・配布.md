# Python 技術ナレッジベース 環境・配布 (Field Manual - Environment & Distribution)
## ■ 本書の役割
`uv` を核としたモダンな環境構築、依存管理、および PyInstaller/Nuitka 等による配布（exe化）に関するナレッジ。**常時ロード必須（04番へ移動）**。

---

## 第1章：配布・環境の絶対防衛 (Survival)

### [K-001] 鉄壁のパス解決
- **絶対座標**: `sys.frozen` 判定を行い、exe内部（_MEIPASS）と外部のパスを厳格に分離せよ。
- **Pathlibの極致**: フォルダ判定は `pathlib.Path.parts` による完全一致判定を徹底せよ。

### [K-002] インポート・ハイジーン (Import Hygiene)
- **パスの聖域化**: エントリポイント（`main.py`）の冒頭において、自身のディレクトリを `sys.path` の先頭に強制挿入するロジック（`sys.path.insert(0, str(Path(__file__).resolve().parent))`）を義務化せよ。
- **配布環境の安定化**: これにより、PyInstaller等による配布環境（exe）や、異なるカレントディレクトリから実行された際でも、app パッケージの読み込み不全（`ModuleNotFoundError`）を根絶せよ。

### [K-003] 高DPIとマルチモニター守護
- **座標バリデーション**: モニター構成変更時にウィンドウが「虚空」へ消えるのを防ぐため、起動時に座標が現在のスクリーン範囲内かチェックせよ。
- **DPI自覚**: `ctypes.windll.shcore.SetProcessDpiAwareness(1)` 等で高解像度環境でのボケを防止せよ。

### [K-015] 標準プロジェクト構成
- **構造の厳守**: `app/` パッケージ化、`pyproject.toml` による一元管理、`.code-workspace` による隔離を徹底せよ。

### [K-016] 機密情報の物理的絶対保護
- **情報流出防止**: `.env` の使用と、Git除外設定（`.gitignore`）を100%徹底せよ。

### [K-017] uv による依存の数学的固定
- **uv.lock の重要性**: `uv lock` により生成される `uv.lock` は、環境の再現性を数学的に保証する「確信の鎖」である。
- **Pythonバージョンの固定**: `.python-version` ファイルをプロジェクトルートに配置し、チーム全員が同一のランタイムを使用することを保証せよ。
- **[K-017-3] バージョン指定の柔軟性**: `.python-version` にはマイナーバージョン（例: `3.11`）までを記述せよ。パッチバージョン（例: `3.11.0`）まで固定すると、`uv` が最新のパッチリリースを認識できず同期に失敗するリスクがある。

### [K-019] uv によるライブラリビルドエラーの回避
- **wheel 優先**: `pip` でビルドに失敗する `pandas` 等のライブラリも、`uv` ならビルド済みバイナリを高速に解決できる。

### [K-042] tkinterweb Plotly表示の依存関係
- **JavaScript有効化**: `HtmlFrame(javascript_enabled=True)` と `pythonmonkey` のインストールが必須。

## 第2章：モダン・エンジニアリング (Modern Tooling)

### [K-050] mypy による静的型検査 (uv 連携)
- **導入の利点**: 実行前に型矛盾を検知し、実行時エラー（TypeError）を劇的に削減する。
- **設定**: `pyproject.toml` の `[tool.mypy]` セクションで管理。
- **実行**: `uv run mypy app/` でプロジェクト全体の型整合性をチェックせよ。

### [K-051] pytest による体系的テスト (uv 連携)
- **テストファースト**: 複雑なロジックは実装前にテストケースを書け。
- **実行**: `uv run pytest` で自動テストを完遂せよ。`tests/` ディレクトリを app/ と同階層に配置するのが標準。

### [K-057] uv による脆弱性スキャン
- **セキュリティ**: `uv` は依存関係の脆弱性を高速にチェックできる。定期的に `uv pip list --outdated` や外部ツールと連携し、クリーンな環境を維持せよ。

### [K-056-1] Loguru による堅牢なロギング設定
- **retention の型**: `logger.add` の `retention` 属性は文字列（例: `"5 generations"`) ではなく、整数値（例: `5`）で指定せよ。ライブラリ内部のパースエラーを物理的に封殺できる。
