# Python 技術ナレッジベース 監査・地雷回避 (Field Manual - Audit & Anti-Mine)

## ■ 本書の役割
過去の失敗と規律から抽出された、実務上の「地雷回避」と「AI能力極大化」のための具体的な戦術集。**常時ロード必須（03番へ上位移動）**。

---

## 第1章：思考の三層監査プロトコル (Triple-Layer Thinking Audit)

ルールの増大に伴う「見逃し」を防ぐため、以下の三段階の思考プロセスを全工程で物理的に踏め。

1. **層1：抽出（Extraction）**
   - 実装・修正に着手する直前、今回のご指示に関連する全ルールとナレッジを脳内で「厳守リスト」としてロードせよ。
2. **層2：写経（Reflection）**
   - 実装中、各コードブロックを記述する度に「この記述はどのルールに基づいているか」を自己対話形式で確認せよ。
3. **層3：検挙（Violation Hunting）**
   - 完了報告の直前、「ルールを守っているか」ではなく、**「ルールを破っていないか」という逆説的な視点（ネガティブ・チェック）**で全層を走査せよ。

---

## 第2章：極限境界値の事前掃討 (Pre-emptive Edge Case Hunting)

「想定外」を抹殺するため、実装前に以下の**「5つの極限シナリオ」**に対する対処方針を明確にせよ。

1. **物理的遮断 (Physical Interruption)**
   - ネットワーク切断、ディスクフル、ファイル使用中（ロック）への耐性。
2. **入力の暴力 (Input Violence)**
   - 巨大なファイル（数GB）、想定外の文字コード、不正なデータ型、空データ。
3. **時間の歪み (Time Distortion)**
   - 処理のタイムアウト、外部APIの無応答、並列処理による競合（レースコンディション）。
4. **権限の壁 (Permission Wall)**
   - 管理者権限不足、読み取り専用ファイル、OSによるアクセス拒否。
5. **環境の衝突 (Environment Conflict)**
   - Pythonバージョンの不一致、他アプリによるポート占有、パスのスペース含有。

## 第3章：AI能力の極大化戦術 (AI Capability Maximization)

AI特有の「推論の揺れ」や「省略癖」を抑制し、達人級の精度を引き出す技術。

### [T-070] 推論の多段再帰化
- 「第1段：解釈」「第2段：自己批判（不備の指摘）」「第3段：再構築（修正案）」を内部的に（または思考プロセスとして）踏め。

### [T-071] 真理値表の物理的記述
- 複雑な `if-else` や条件分岐を実装する際、すべての組み合わせを網羅した「真理値表」を Docstring 内に物理的に記述せよ。

### [T-072] 物理的計量スクリプトの実行
- 文字数・行数差分の報告において、AIの脳内カウントではなく、以下のロジック（または実測値）を基準とせよ。
  ```python
  # 実測基準
  lines = len(file_content.splitlines())
  chars = len(file_content)
  ```

## 第4章：物理的ネガティブ監査リスト (Negative Check List)

納品直前、以下の「不潔なパターン」が存在しないか物理的にスキャンせよ。

- **知的負債の残存**: `TODO:` `FIXME:` `とりあえず` `あとで`。
- **構成の旧式化**: `pyproject.toml` や `uv.lock` の欠落、`pip`/`venv` への直接依存。
- **自動監査の欠落**: `ruff` や `mypy` の設定が `pyproject.toml` に含まれていない。
- **論理的断絶**: 仕様書のレビジョンがコードと不一致。ADRへの記録漏れ。
- **情報の抹消**: `(中略)` による、ユーザーが意図しないコードの消去。
