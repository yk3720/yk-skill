# プログラム実行ファイル（EXE）化

### PyInstallerを使用する方法
1. インストール
   ```bash
   pip install pyinstaller
   ```
2. 実行ファイル作成（1つのファイルにまとめる場合）
   ```bash
   pyinstaller --onefile script.py
   ```
   - `--noconsole`：実行時にコンソール画面を表示させない場合に追加
   - `-i icon.ico`：アイコンを指定する場合に追加

#### メリット
- **標準的**: 最も広く使われており、情報が豊富。
- **自動化に適している**: コマンドラインで完結するため、バッチ処理やCI/CD（自動ビルド）に組み込みやすい。
- **詳細設定**: `.spec` ファイルを編集することで、非常に細かいビルド設定が可能。

#### デメリット
- **コマンド操作**: 初心者にはコマンドのオプション指定が難しく感じることがある。
- **環境依存**: ビルドしたOSと同じOSでしか動かない（例：WindowsでビルドしたものはWindowsでしか動かない。※これはauto-py-to-exeも共通）。

### auto-py-to-exeを使用する方法（GUI）
1. インストール
   ```bash
   pip install auto-py-to-exe
   ```
2. ツール起動
   ```bash
   auto-py-to-exe
   ```
   ※ブラウザでGUIが立ち上がり、設定を視覚的に行えます。

#### メリット
- **直感的**: GUI上でファイル選択やアイコン指定ができるため、ミスが少ない。
- **コマンド生成機能**: GUIで設定した内容を PyInstaller のコマンドとして表示してくれるため、学習にも役立つ。
- **ファイルの追加が楽**: 外部データ（画像やCSVなど）を同梱する設定が視覚的に行いやすい。

#### デメリット
- **追加の依存関係**: PyInstaller に加えてこのツール自体をインストールする必要がある。
- **自動化に不向き**: 手動操作が前提のため、大量のスクリプトを自動でEXE化するのには向かない。
- **中身はPyInstaller**: 結局内部で PyInstaller を動かしているだけなので、PyInstaller特有の不具合や「EXEが重い」「ウイルスソフトに誤検知される」といった問題は解消されない。
