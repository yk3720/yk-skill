## ■ 標準構成：達人級・一発必中構成

配布成功率100%に加え、`uv` による爆速かつ厳密な環境構築を統合した究極の構成。

```
project_root/                # 【ビルド環境】開発者のPCにのみ存在
├── 仕様・管理/               # 【最重要】アプリ固有の文脈
│   ├── プロジェクト仕様書.md   # 唯一の真実（What）
│   ├── 意思決定記録(ADR).md  # なぜその設計にしたか（Why）
│   ├── テスト定義書.md       # 正解の基準（Check）
│   └── [廃止] 取扱説明書.md (プロジェクト仕様書へ統合)
├── pyproject.toml           # 【真実の核】uv/Ruff/ビルド設定の一元管理
├── uv.lock                  # 【確信の鎖】uv による厳密な依存関係の固定
├── main.py                  # 【エントリポイント】インポート・ハイジーン実装
├── setup_venv.py            # 【環境構築】uv をバックエンドに使用
├── build_exe.py             # 【ビルドスクリプト】
├── clean.py                 # 【浄化スクリプト】
├── ビルド実行.bat            # 【聖なる引き金】浄化とビルドを連続実行
├── requirements.txt         # 【下位互換】uv export で自動生成
├── .python-version          # Pythonバージョンの固定
├── .gitignore               # Git除外リスト
├── .venv/                   # 仮想環境（Git除外対象）
├── .vscode/                 # IDE設定（settings.json, launch.json）
├── プロジェクト名.code-workspace  # 【最重要】ワークスペース隔離ファイル
├── assets/                # 【アセット】アイコン、画像等（1KB以上）
├── logs/                  # 【ログ】実行ログの隔離場所
├── output/                # 【出力先】
├── temp/                  # 【一時ファイル】即時清算対象
└── app/                  # ソースコード本体（500行制限を遵守）
    ├── __init__.py
    ├── constants.py         # 定数定義
    ├── core/                # 主要ロジック
    ├── ui/                  # GUI定義
    └── utils/               # 共通機能・パス解決
```

### 配布成功のための鉄則

1. **uvによる環境統治**: `pip` に代わり `uv` を採用し、環境構築の不確実性と時間を極限まで削減する。
2. **pyproject.tomlの一元管理**: メタデータ、依存関係、Ruff設定、ビルド設定を一つのファイルに集約する。
3. **パス解決の厳格化**: `sys.frozen` を判定し、exe内部と外部のパスを正しく切り分ける。
4. **アセットの埋め込み**: 1KB未満はBase64、1KB以上は `assets/` への隔離を徹底する。
5. **自己修復機能**: 必要なディレクトリは起動時に自律生成する。
6. **インポート・ハイジーン**: `main.py` 冒頭での `sys.path` 初期化を義務化する。
7. **`.code-workspace`による隔離**: ワークスペースルートをプロジェクト内に限定し、仮想環境の混線を物理的に防ぐ。

## ■ ファイル構成に込めたエンジニアリング哲学

### 1. 「一発必中」の具現化
達人にとって、配布先での起動失敗はエンジニアリングにおける「恥辱」である。`uv.lock` による厳密な固定は、この「一発必中」を数学的に保証する。

### 2. ソースコードの「聖域化」
ソースコードを `app/` に封印し、1ファイル500行制限を課すことで、論理の純度を守り、AIによる「中略」を構造的に排除する。

### 3. 「生存能力（サバイバビリティ）」
`uv` による爆速環境構築は、開発者の入れ替わりやOSの刷新といった過酷な状況下でも、プロジェクトが即座に息を吹き返す「再生能力」を意味する。
