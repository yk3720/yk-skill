# M2 更新 — 手順（v1 骨格）

**ステータス:** v1 は骨格のみ。本ファイルを Read したらユーザーに「M2 は v1 未実装。整合（M1）を先に」と伝えてよい。

## 目的

資料の **中身** を Web 最新情報と照合し、冗長・旧バージョン記述を整理する。M1（資料同士の矛盾）とは別モード。

## 将来の手順（予定）

1. 対象ファイルをユーザーまたは Tier P で特定
2. **`researching-web` を Read して委譲** — 公式ドキュメント優先
3. 差分表 — **外部事実** vs **プロジェクト決定** を分離（外部を資料に上書きしない）
4. 旧情報処理 — **アーカイブ優先**（`docs/99_アーカイブ/` または `PROJECT_DOCUMENT_RULES` §5）
5. 完全重複段落のみ削除可 · 方針変更で古くなった章は移動
6. 報告 · **commit しない**

## ゲート（予定）

| 操作 | 要否 |
|------|------|
| ファイル削除 | 一覧提示 → ユーザー OK |
| アーカイブ移動 | 機械的重複は可 · それ以外は OK |
| 外部 Web 情報の取り込み | 出典 URL 必須 |

## v1 でやること

M2 依頼が来た場合:

1. 本ファイルと [scope-tiers.md](scope-tiers.md) を Read
2. 対象範囲を 1 問確認
3. **M1 で整合が取れているか**を軽く確認（取れていなければ M1 を提案）
4. 実装完了まで **調査結果の報告のみ** でも可
