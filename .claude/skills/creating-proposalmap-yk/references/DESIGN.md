# ProposalMap — 設計メモ（SSOT）

**形式 id:** `proposalmap`  
**日本語名:** 社内提案図解（周知用）  
**スキル:** `creating-proposalmap-yk`  
**策定:** 2026-05-24（対話・Web調査・サブエージェントレビュー反映）

---

## 1. 目的と読者

| 項目 | 内容 |
|------|------|
| **主目的** | 「**こんなものを作ろうとしています**」を関係者に共有する（**周知**） |
| **含まない** | 決裁・予算・投資の正式依頼（冒頭で明示して誤解を防ぐ） |
| **読者** | 社内の非IT部門（管理職・業務担当など） |
| **文体** | **です・ます**調。平易だがカジュアルすぎない（スラング・過度なたとえ・「！」乱用禁止） |
| **体裁** | 企画書・提案書の章立て（HTML 図解 · surge 公開） |

---

## 2. 既存形式との境界

| 形式 | 用途 | proposalmap との差 |
|------|------|-------------------|
| curiositymap | 文系・日常起点・学習の入口 | たとえ話・疑問形中心。社内周知・境界表は弱い |
| techmap | 理系エンジニア向け技術解説 | 技術深掘り。非専門の社内説明には不向き |
| visual | 読者未指定の汎用図解 | 企画・提案の型・周知固定文がない |
| **proposalmap** | 社内・非専門・作ろうとしているものの周知 | 結論ファースト・やる/やらない・お願いの種類を固定 |

---

## 3. 必須ブロック（HTML 章順）

| 順 | id | 見出し（例） | 内容 |
|----|-----|-------------|------|
| 0 | notice | （帯） | 周知用・決裁依頼ではない旨（固定文） |
| 1 | summary | 3分で把握 | 5行以内：何を／なぜ／どうする／境界1行／お願い |
| 2 | background | 背景と課題 | 業務の痛み（定量があれば1つ） |
| 3 | purpose | 目的とゴール | 達成したい状態（非専門語） |
| 4 | solution | 作ろうとしているもの | 業務語で仕組み＋簡易フロー図。技術語は補足 |
| 5 | scope | できること／今回しないこと | 2列表。しない理由を1語で |
| 6 | status | 進め方・現状・次の一手 | MVP/Phase・リスク最大3件 |
| 7 | terms | 用語（本文に出た語のみ） | 最大5語 |
| 8 | closing | お願い | 「ご覧ください」「ご意見歓迎」等（周知用） |

**固定文（notice）:**

> 本資料は、現在検討・開発中の取り組みを関係者に共有するための**周知用**説明です。決裁・予算のご依頼を含みません。

---

## 4. トーン・用語

- **プレーンディスク:** 情報を削らず、見出し・表・初出定義で明確化（SEC / CAN-ASC-3.1 準拠の考え方）
- **専門用語:** 初出で1行定義。本文は**業務語を先**、技術語は括弧補足
- **お願いの種類（周知）:** 承認｜周知｜コメント依頼 — proposalmap 既定は **周知**
