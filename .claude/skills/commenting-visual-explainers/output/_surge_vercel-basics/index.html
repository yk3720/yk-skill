ここから先で、Vercel が何をしているかをひとつずつ解説していきます。

Vercel って何？

Vercel（バーセル）は、Webサイトやアプリを「インターネットに公開する」作業をまるごと肩代わりしてくれるサービスです。Next.js の開発元が作っており、2026年現在も世界で最も広く使われるデプロイプラットフォームの一つです。

たとえ話で考えてみましょう。あなたが手紙（コード）を書いたとします。その手紙を「封筒に入れて → 宛名を書いて → 郵便局に持って行く」作業が、これまでのWeb公開でした。Vercelはその一切をやってくれる宅配業者です。手紙を渡すだけで、世界中の相手に届けてくれます。

ここがポイント

以前のWeb公開は「サーバーを借りて → Node.jsを設定して → ビルドして → ファイルを転送して…」と多くの手順が必要でした。Vercelはこれをゼロコンフィギュレーション（設定ほぼなし）で実現します。

「git push」から公開まで何が起きているか

「git push したら公開された」── この一言の裏側で、Vercelは5つのステップを自動で実行しています。

Vercelが自動で実行する5ステップ

1
検知（トリガー）

GitHubへの push や PR のマージを Webhook で即座に検知。「コードが変わった」と判断した瞬間、次のステップが自動で始まります。

2
フレームワーク自動検出

Next.js・Astro・SvelteKit など、何で作られているかを自動判定し、最適なビルド設定を自動適用。設定ファイルを書く必要はありません。

3
ビルド

npm run build を隔離された仮想マシン上で実行。TypeScript のコンパイル・画像の最適化・HTMLの生成など、公開に必要な変換をすべて行います（30秒〜2分程度）。

4
世界中のサーバーへ配置

ビルド成果物を 100か所以上の拠点（PoP）に自動配置。日本のユーザーには日本のサーバーから、アメリカのユーザーにはアメリカのサーバーから届くため、表示が速くなります。

出典: Vercel 公式ドキュメント — How requests flow

5
URL が発行される

デプロイが完了すると、本番URL（xxx.vercel.app） が即座に使えるようになります。直前まで古いバージョンが表示されていて、完了と同時に新しいバージョンに切り替わります（ダウンタイムなし）。

3つの「環境」── 本番・プレビュー・ローカル

Vercel には3種類の環境があります。「本番」「プレビュー」「ローカル」です。それぞれ役割が違い、うまく使い分けることで「本番に間違いを出さない」開発ができるようになります。

Production（本番）

実際にユーザーが見る「本物」のサイト。main ブランチへのpushで更新される。

xxx.vercel.app
Preview（プレビュー）

PR（プルリクエスト）ごとに自動で別URLが発行される仮の環境。「本番に出す前に確認」する場所。

xxx-git-branch.vercel.app
Development（ローカル）

自分のPCの中だけで動く開発用環境。npm run dev で起動し、外からは見えない。

localhost:3000

このツール（図解コメントツール）でVercelがやっていること

あなたが今日セットアップした「図解コメントツール」では、Vercelはコメントデータを受け取って保存・取得するAPIサーバーとして動いています。

commenting-visual-explainers-liart.vercel.app
Vercel上で動いているもの
/api/comments
コメントの保存・取得・削除を行うAPIエンドポイント。Neon（データベース）と繋がっている
/widget.js
図解ページに埋め込まれるフィードバックUIのJavaScriptファイル。世界中から高速に配信される
API_TOKEN による保護
セットアップ時に生成したトークンが環境変数として設定済み。不正アクセスを防いでいる

なぜVercelを使うのか

コメントAPIを動かすには「サーバー」が必要です。自分でサーバーを立てると維持費・管理の手間がかかりますが、Vercel なら無料で・管理なしで・世界中から使えるAPIサーバーが手に入ります。図解ページがどこに公開されていても、コメントは常にこのURLに届きます。

まとめ ── 覚えておきたい3つのこと

01

Vercelは「Web公開の宅配業者」

git push するだけで、ビルド・世界中への配置・URLの発行まですべて自動でやってくれます。サーバーの知識は不要です。

02

3つの環境を使い分ける

ローカル（自分のPC）→ プレビュー（確認用）→ 本番（公開）の3段階で、本番に間違いを出さない開発ができます。

03

無料で世界規模のインフラが使える

個人利用なら完全無料。100か所以上の拠点から高速配信・自動HTTPS・ゼロダウンタイム更新が、設定なしで手に入ります。

「公開する」の敷居がなくなる、それがVercelです。

以前は「サーバーを借りて・設定して・デプロイして…」という複雑な作業が必要でした。Vercel はその作業を消し去り、「作ることに集中する」開発者体験を実現しました。今回セットアップした図解コメントツールも、この仕組みの上で動いています。
