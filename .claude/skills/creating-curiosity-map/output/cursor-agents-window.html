AI が自動でコードを書く時代

Cursor Agents Window とは何か

「プログラミングがよくわからなくても大丈夫」な入門図解

ひとことで言えば

Cursor Agents Window とは、複数の AI アシスタントに同時に仕事を頼んで、その進み具合を一画面で見渡せる「指揮センター」のようなものです。

Cursor（カーソル）はプログラマーが使う AI つきコードエディタです。2026年4月に大きなバージョンアップがあり、「Agents Window（エージェントウィンドウ）」と呼ばれる新しい画面が登場しました。この画面を使うと、AI に複数の作業を同時に任せながら、それぞれの進捗をまとめて確認できるようになりました。

エージェント（Agent）: ここでは「AI が自律的に動いて作業を進める存在」のことです。人間が細かく指示しなくても、ゴールを伝えるだけで自分でファイルを調べ、変更し、結果を返してくれます。

なぜ今、注目されているのか

プログラマーの仕事は、ここ数年で大きく変わりつつあります。以前は「自分でコードを1行ずつ書く」のが当たり前でしたが、AI が進化したことで「AI に書いてもらい、人間はレビューする」という流れが定着し始めました。

1

「AI チャット」から「AI エージェント」へ

従来の ChatGPT のような AI は「質問して答えを返してもらう」ものでした。しかし最新のエージェント型 AI は「ゴールを伝えると、自分でファイルを読み書きしながら課題を解決する」ところまで動きます。

2

1人で複数の AI アシスタントを動かせる時代に

Agents Window では、複数の AI アシスタントを同時に起動して、それぞれ別の仕事をさせることができます。「テスト担当」「ドキュメント担当」「バグ修正担当」のように役割を分けて、並行して動かすようなイメージです。

3

「プログラマーが手を動かす時代」から「AI を監督する時代」へ

Cursor 3 のリリース発表では「ファイルを編集する人からエージェントを指揮する人へ」という表現が使われました。プログラマーの役割が、職人から監督・ディレクターへと変化しつつあります。

Agents Window はこの変化を実現するための画面として、2026年4月2日（日本時間）にリリースされました。

まず理解する3つ

この3つがわかれば、会話に参加できるレベルになります。

1. 複数の AI に同時に仕事を頼める

「テストを書いて」「バグを直して」「ドキュメントを更新して」——これをひとりの AI に順番にやらせると時間がかかります。Agents Window では、これを別々の AI アシスタントに同時に頼むことができます。4人分の仕事を30分で終わらせることも可能です。

2. パソコンを閉じても AI が働き続ける

「クラウドエージェント」と呼ばれる機能を使うと、大きな仕事をインターネット上のサーバーで動かすことができます。パソコンの電源を切っても作業は続いており、数時間後にパソコンを開くと結果が出来上がっています。夜のうちに仕事を任せて、朝に確認するイメージです。

3. 各 AI に専用の「作業部屋」を与える

複数の AI が同じファイルを同時に書き換えると、変更がぶつかってしまいます。Agents Window では「ワークツリー」という仕組みを使い、各 AI に独立した作業部屋（ファイルのコピー）を与えることができます。お互いの作業を邪魔せず、失敗したら部屋ごと捨てられます。

たとえ話 1: 「複数のアシスタントへの同時依頼」

大きな引っ越しを考えてください。1人で全部やるより、「梱包係」「掃除係」「手続き係」に分けてお願いした方が早く終わります。Agents Window はこれと同じで、「テスト担当 AI」「バグ修正 AI」「ドキュメント AI」に別々の仕事を同時に頼むことができます。人間の指揮官（あなた）は進捗を確認しながら、必要に応じて指示を変えるだけです。

たとえ話 2: 「部屋ごとに分かれた作業場（ワークツリー）」

料理教室で同じレシピを複数の生徒が同時に作るとき、同じまな板を取り合うと混乱します。「ワークツリー」はそれぞれの生徒に専用のまな板と食材を用意するようなものです。失敗した生徒の食材はそのまま捨てられ、成功した生徒の料理だけが最終的な食卓に並びます。

たとえ話 3: 「工場の夜間シフト（クラウドエージェント）」

工場では昼間に作業員が働き、夜はロボットが無人で生産を続けます。クラウドエージェントはこの「夜間ロボット」に相当します。パソコンを閉じてもインターネット上のサーバーで AI が働き続け、翌朝には完成品（コードの変更）が出来上がっています。大きな仕事ほど夜間に任せると効率的です。

たとえ話 4: 「設計図のチェック（プランモード）」

リフォーム工事では、職人が工事を始める前に設計図を施主に見せて確認してもらうことが普通です。「プランモード」はこれと同じで、AI が実際に作業を始める前に「こういう手順で進めますが、よいですか？」と計画書を出してきます。方向性が違ったら、この段階で修正できるので、後から「こんなつもりじゃなかった」が防げます。

Agents Window の基本的な使い方の流れ

開

Ctrl+Shift+P → 「Open Agents Window」

Mac は Cmd+Shift+P。コマンドパレットという画面から開きます。

計

複雑なタスクは「プランモード」で計画を確認

入力欄で「Shift+Tab」（Shift を押しながら Tab キー）を押すとモード切り替え。AI が設計書を出したら読んで「OK」か修正を伝える。

頼

ゴールを日本語で伝えて任せる

「ログイン機能を追加して」のようにゴールだけ伝えれば、AI が自分でファイルを読んで作業します。

確

変更内容（差分）を確認して承認 or 却下

AI が変更したのはどこか？がわかりやすく表示されます。問題なければ承認してコードに反映。

よくある誤解・思い込み

「AI が完全に自律して何でもやってくれる」

実際には、AI はジュニアエンジニア（経験の浅い担当者）のようなものです。具体的でスコープの決まった仕事（テストを書く・ドキュメントを更新するなど）は得意ですが、あいまいな要件や複雑な設計判断はまだ人間のチェックが必要です。「全部お任せ」ではなく「任せて確認する」が現実的な使い方です。

「Agents Window はエンジニアにしか関係ない話だ」

確かに実際に操作するのは主にエンジニアです。しかし「AI に仕事を並列で頼める」「PC を閉じても作業が続く」という仕組みを知ることは、プロジェクトの依頼・コスト見積もり・スケジュール感の共有において非エンジニアにも大いに役立ちます。道具の存在を知らずにいると、「なぜエンジニアがこんなに早く仕上げられるのか」も「なぜコストがかかるのか」も見えなくなります。

「Agents Window が使えると ChatGPT は要らなくなる」

Agents Window はあくまで「コーディング作業」に特化したツールです。ChatGPT のような汎用 AI とは目的が異なり、「コードのファイルを操作して実際の変更を行う」という点が最大の違いです。両者は補完関係にあります。

「AI に任せれば多いほどよい」

並列で動かすエージェントが多いほどコストがかかります。また、確認が増えすぎると逆に管理が大変になります。「何を AI に任せ、何は自分でやるか」を選ぶ判断眼こそが、これからの時代に求められるスキルです。

知っておきたい言葉

Cursor（カーソル）
AI 機能を組み込んだコードエディタ。Visual Studio Code（VS Code）をベースに作られており、AI がコードの提案・生成・修正を行ってくれる。
エージェント（Agent）
ゴールを伝えると自律的に動いて仕事を進める AI のこと。ファイルを読んで書き換えたり、エラーを修正したりと、単なる「回答する AI」より能動的に動く。
クラウドエージェント / バックグラウンドエージェント
インターネット上のサーバーで動くエージェント。パソコンを閉じても動き続け、数時間後に結果を返してくれる。「夜間シフト」のように長時間の作業に向いている。→ エージェントと対比。
ワークツリー（Worktree）
各エージェントに割り当てる「専用の作業部屋（ファイルのコピー）」。別のエージェントと作業がぶつかるのを防ぐ。失敗しても部屋ごと捨てられる。
差分（Diff）
AI が変更した箇所だけをわかりやすく表示したもの。「ここを追加した」「ここを削除した」が色分けで見える。全文を読まなくても変更点の確認ができる。
プランモード（Plan Mode）
AI が実際に作業を始める前に計画書を出してくれるモード。方向性を確認してから作業に入るため、「やり直し」が減る。複雑な依頼をするときに有効。
デザインモード（Design Mode）
ウェブサイトやアプリの画面（見た目の部分）を見ながら「このボタン」「この文字」と直接指定して AI に修正を頼めるモード。言葉だけで場所を説明しなくてよいので直感的に使える。
※「フロントエンド」= 利用者が目にする画面側のこと。ボタン・色・レイアウトなどの見た目部分。これに対し「バックエンド」はデータ処理など裏側の仕組みを指す。
PR（Pull Request：プルリクエスト）
コードの変更を本番環境に反映する前に、チームメンバーにレビューを依頼する仕組み。Agents Window ではこの管理画面も内蔵されており、AI が作った変更をそのまま PR として提出できる。
バイブコーディング（Vibe Coding）
「雰囲気でコードを書く」という造語。AI に「なんとなくこんな感じ」とゴールだけ伝えて、細かい実装は AI に任せるスタイルのこと。Agents Window はこのスタイルをより洗練させたもの。

もっと知りたい人へ

AI エージェント全般の進化

OpenAI・Anthropic・Google など各社が開発するエージェント型 AI の最新動向。コーディング以外の仕事への応用も急拡大中。

GitHub との連携

コードの変更履歴管理サービス「GitHub」と Cursor の連携。PR の自動作成・レビューの自動化など、チーム開発での活用方法。

Slack / Linear との連携

チャットツール（Slack）やタスク管理ツール（Linear）から Cloud Agent を操作できる機能。「スマホから AI に仕事を頼む」体験が実現しつつある。

ソフトウェア開発の未来

AI がコードのほとんどを書くようになった場合、エンジニアの仕事はどう変わるか。「設計・レビュー・判断」に特化する時代の働き方。
