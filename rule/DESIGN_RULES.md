## 1. デザインの 4 原則（CRAP）

Robin Williams の著書 *The Non-Designer's Design Book* で提唱された 4 つの原則。  
「CRAP」は頭文字を並べたもの（否定的な意味ではない）。

### C — Contrast（対比）
> 要素が同じでないなら、はっきり違うように見せる。

- 重要なもの（件名）と補助的なもの（起票日）を、**フォントサイズ・ウェイト・色**の組み合わせで明確に区別する
- セクションヘッダーはダーク背景＋白文字で、本文エリアと強烈なコントラストをつける
- 期限超過行を `bg-rose-50` で背景着色し、通常行と対比させる

このレポートでの適用：
- 件名 → `font-weight: 500` + `color: #1e293b`（最暗）
- 設備名 → `color: #475569`（中間）
- 日付・No → `color: #64748b`（薄め）
- バッジ → 白文字ではなく**濃テキスト＋淡背景**で視認性向上

### R — Repetition（反復）
> デザインの視覚的要素をすべての資料で繰り返す。統一感が生まれる。

- 全テーブルの縞模様色を `#f8fafc`（slate-50）に統一（旧実装では定期: gray-50 / 不具合: orange-50 と異なっていた）
- セクション間のマージンを全て `16px`（`--sp-2`）に統一
- テーブルセルの padding を全て `8px 16px`（`--sp-1 --sp-2`）に統一
- バッジの形状：ステータス系 = pill、優先度・種別 = rounded square と**形状で役割を区別**し、グループ内は完全同形状

### A — Alignment（整列）
> ページ上のすべての要素を別の要素と視覚的に結びつける。

- **左辺基準線**: ヘッダー・カード・テーブルすべての左内 padding を `16px` に統一
- **数値の右揃え**: テーブルの No 列は `text-align: center`（数値は中央または右揃えが自然）
- バッジ内テキストを `display: inline-flex; align-items: center` で縦方向に中央揃え
- サマリーカードの数値に `font-variant-numeric: tabular-nums` を指定し、数値変動時にレイアウトがずれない

### P — Proximity（接近）
> 関連する要素は近くにまとめ、無関係な要素とは離す。

- サマリーカード 4 枚を `gap: 16px` のグリッドで密に配置（ひとつのグループとして認識）
- フィルターラベルとボタン群を `gap: 8px` で近接（「装置：」と装置ボタンが一体に見える）
- ガントチャートの凡例アイコンとラベルを `gap: 4px` で密接させる
- セクションヘッダーと直下のコンテンツの間隔を 0 にして、ヘッダーとコンテンツが一体に見える

## 2. 8px グリッドシステム

### 考え方

すべての**スペーシング**（margin / padding / gap / line-height）を **8 の倍数**で定義する。  
これにより要素間の余白が自然に揃い、計算も単純になる。

4px は 8px グリッドの「サブグリッド」として、バッジ内余白など極細の用途にのみ許容する。

### トークン定義

| トークン | px | 主な用途 |
|---------|-----|---------|
| `--sp-0` | 4px | バッジ内 padding・凡例アイコンとラベルの間 |
| `--sp-1` | 8px | テーブルセルの縦 padding・コンポーネント内の最小間隔 |
| `--sp-2` | 16px | カード内 padding・セクション間マージン・ボタン横 padding |
| `--sp-3` | 24px | ページ余白（モバイル） |
| `--sp-4` | 32px | ページ下部余白 |
| `--sp-6` | 48px | ページ余白（デスクトップ） |

### 禁止値の例

以下の値は 8px グリッド外なので**使用しない**：
- `10px`（→ `8px` or `16px` に変更）
- `6px`（→ `8px` に変更）
- `20px`（→ `16px` or `24px` に変更）
- `12px`（→ `8px` or `16px` に変更）※ テーブルセルの `px-3` は `px-4` に変更

## 3. タイポグラフィスケール

### 考え方

- **font-size** は 8px 倍数に縛らない（14px, 13px, 15px 等は許容）
- **line-height は必ず 8px の倍数**（16px / 24px / 32px）に揃える  
  → テキストの高さが 8px グリッドに乗るため、他の要素と高さが揃いやすくなる
- 日本語 UI では英語 UI より少し広めの line-height が読みやすい（body は 24px 推奨）

### スケール定義

| スケール名 | font-size | line-height | font-weight | CSS クラス | 用途 |
|----------|-----------|-------------|-------------|-----------|------|
| **display** | 24px | **32px** | 700 (bold) | `.t-disp` | サマリーカードの件数数値 |
| **heading** | 15px | **24px** | 600 (semibold) | `.t-head` | ページタイトル・セクション見出し |
| **body** | 14px | **24px** | 400 (regular) | `.t-body` | テーブル本文・標準テキスト |
| **small** | 13px | **16px** | 400 (regular) | `.t-sm` | 状況・結果など長文折返し列 |
| **label** | 12px | **16px** | 500 (medium) | `.t-lbl` | バッジ・フィルターボタン・補助テキスト・日付 |

### フォントスタック

```css
font-family: system-ui, 'YuGothic', 'Yu Gothic', 'Hiragino Sans', 'Noto Sans JP', sans-serif;
```

- `system-ui`: OS のデフォルトフォントを使用（高速）
- `YuGothic` / `Yu Gothic`: Windows 日本語環境向け
- `Hiragino Sans`: macOS / iOS 日本語環境向け
- `Noto Sans JP`: Android・Linux 向けフォールバック

### font-variant-numeric: tabular-nums

サマリーカードの数値（件数）に `tabular-nums` を指定する。  
これにより、数値が変化しても文字幅が変わらず、レイアウトが崩れない。

## 4. カラーシステム

### 考え方：彩度と視認性

**彩度（Saturation）** とは色の鮮やかさ。  
ビジネス UI では過度に高彩度の色は視覚的疲労を引き起こす。

| 彩度 | 用途 | 例 |
|-----|------|---|
| 高彩度（100%） | アラート・エラー・ CTA ボタン | 危険な警告 |
| 中彩度（40〜60%） | セクションヘッダー | このレポートのヘッダー |
| 低彩度（10〜20%） | バッジ背景・テーブル縞 | chip スタイルバッジ |
| 無彩色（0%） | テキスト・ボーダー | slate 系グレー |

### WCAG 2.1 AA 基準

Web Content Accessibility Guidelines で定められたコントラスト比の最低基準。

| テキストサイズ | 最低比率 |
|------------|---------|
| 通常テキスト（< 18px） | **4.5:1** |
| 大きいテキスト（≥ 18px、または bold ≥ 14px） | **3:1** |

> 旧実装では `text-orange-500`（#f97316）の白背景コントラスト比が 2.68:1 で不合格。

### テキストカラー（slate 系）

| トークン | hex | コントラスト比（白背景） | 用途 |
|--------|-----|----------------------|------|
| `--c-t1` | `#1e293b` | **15.6:1** ✓ | 件名・主要テキスト |
| `--c-t2` | `#475569` | **7.4:1** ✓ | 設備名・状況・副テキスト |
| `--c-t3` | `#64748b` | **4.7:1** ✓ | 日付・No・補助テキスト |
| `--c-t4` | `#94a3b8` | 2.85:1（装飾のみ） | ダッシュ（—）など装飾的テキスト |

### ページ・カラーパレット

| トークン | hex | 用途 |
|--------|-----|------|
| `--c-page` | `#f1f5f9` | ページ背景（青みがかったグレー = 文書的印象） |
| `--c-card` | `#ffffff` | カード背景 |
| `--c-stripe` | `#f8fafc` | テーブル縞（全テーブル共通） |
| `--c-bd` | `#e2e8f0` | 標準ボーダー |
| `--c-bds` | `#94a3b8` | 強調ボーダー |

### セクションヘッダー（彩度抑制）

旧実装（高彩度）から、色相を維持しつつ彩度を約 15〜20% 抑制。  
同じ色相ファミリーで区別することで「違うセクション」と「同じシステム」を両立。

| セクション | ヘッダー bg | thead bg | 色相 |
|----------|-----------|---------|-----|
| ガントチャート | `#1e3a5f` | `#2d5080` | 青系ネイビー |
| 定期作業 | `#1c4532` | `#276046` | 緑系ダーク |
| 不具合・特記事項 | `#5c2d1a` | `#7a3d25` | 茶系ダーク |

### バッジ: Chip スタイル

旧実装（白文字＋濃色背景）からすべて **淡背景＋濃テキスト＋ボーダー** の chip スタイルへ変更。  
すべて **7:1 以上**のコントラスト比を確保。

#### ステータスバッジ（pill: border-radius: 9999px）

| ステータス | bg | text | border | コントラスト比 |
|----------|----|----|---------|-----------|
| 完了 / 済 | `#dcfce7` | `#14532d` | `#86efac` | 9.2:1 ✓ |
| 対応中 | `#fef3c7` | `#78350f` | `#fcd34d` | 7.3:1 ✓ |
| 確認中 | `#f3e8ff` | `#581c87` | `#d8b4fe` | 8.3:1 ✓ |
| 予定 | `#f1f5f9` | `#1e293b` | `#cbd5e1` | 15.6:1 ✓ |

#### 優先度バッジ（rounded square: border-radius: 4px）

形状をステータスバッジ（pill）と変えることで、同じ列に並んでも役割が区別できる。

| 優先度 | bg | text | border | コントラスト比 |
|-------|----|----|---------|-----------|
| 高 | `#fee2e2` | `#7f1d1d` | `#fca5a5` | 8.0:1 ✓ |
| 中 | `#fff7ed` | `#7c2d12` | `#fed7aa` | 8.7:1 ✓ |
| 低 | `#f8fafc` | `#475569` | `#e2e8f0` | 7.4:1 ✓ |

#### 種別バッジ（rounded square: border-radius: 4px）

「不具合」と優先度「中」が同色系になるため、**左ボーダーを橙色（#ea580c）に**して区別。

| 種別 | bg | text | border | 特記 |
|-----|----|----|---------|-----|
| 不具合 | `#fff7ed` | `#7c2d12` | `#fed7aa` | 左ボーダー: `#ea580c` で識別 |
| 特機事項 | `#eef2ff` | `#312e81` | `#a5b4fc` | — |

#### サマリーカード数値

| カード | hex | コントラスト比（白背景） |
|------|-----|----------------------|
| 定期 完了 | `#15803d`（green-700） | 4.56:1 ✓ |
| 定期 進行中 | `#0369a1`（sky-700） | 5.91:1 ✓ |
| 不具合 対応中 | `#c2410c`（orange-700） | 5.65:1 ✓ |
| 不具合 確認中 | `#6b21a8`（purple-800） | 5.29:1 ✓ |

## 5. CSS 変数の命名規則

```
--sp-{n}     スペーシング（n × 8px）
--fs-{scale} フォントサイズ（スケール名）
--lh-{scale} ライン高さ（スケール名）
--c-{name}   カラー（汎用）
--c-t{1-4}   テキストカラー（階層）
--c-bd, --c-bds  ボーダーカラー
--c-g*, --c-p*, --c-t*h  セクションカラー（g=gantt, p=planned, t=trouble）
--s*-{bg/c/bd}   ステータスバッジカラー
--p*-{bg/c/bd}   優先度バッジカラー
--k*-{bg/c/bd}   種別バッジカラー
--rod-{bg/hv/c}  期限超過行カラー
--sum-*          サマリー数値カラー
```

## 6. コンポーネントルール一覧

### Card（`.card`）
- `border-radius: 8px`
- `border: 1px solid var(--c-bd)`
- `overflow: hidden`（子要素がはみ出ないよう）
- `box-shadow`: 極めて薄い影（0 1px 2px）

### Table（`.rpt-tbl`）
- セル padding: `var(--sp-1) var(--sp-2)`（8px 16px）
- thead の行は列ごとに `.ph` / `.th` クラスで色を制御
- 縞模様: 奇数行 `var(--c-stripe)`、偶数行（デフォルト）= 白
- 期限超過行: `.od` クラスを付与

### Badge（`.bdg`）
- 基本: pill（border-radius: 9999px）
- 優先度・種別: `.sq` 追加で rounded square（border-radius: 4px）
- padding: `var(--sp-0) var(--sp-1)`（4px 8px）

### Filter Button（`.filter-btn`）
- アクティブ: `background: var(--c-gh)`（ガントチャートヘッダーと同色でフィルター対象と連動を示す）
- pill 形状

## 7. このシステムの拡張ルール

新しい要素を追加する際の判断基準：

1. **スペーシング**: 8px グリッドの値を使う。迷ったら `--sp-1`（8px）か `--sp-2`（16px）
2. **テキスト色**: 新しい色は使わず `--c-t1` 〜 `--c-t3` から選ぶ
3. **バッジ**: 新しいステータスは chip スタイルを踏襲し、コントラスト比 7:1 以上を確認
4. **セクションヘッダー**: 新セクションは色相を変えるが、明度・彩度は既存と揃える
5. **フォント**: スケール外のサイズは追加しない。用途が近いスケールに寄せる
